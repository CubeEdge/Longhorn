1. **强制执行 `/upd` 流水线**:
   - 递增 `client/package.json`（和必要的 `server/package.json`）中的最后一位 (Z位) 版本号。
   - 运行 `npm run build` 确保前端代码编译成功。
   - 运行 `./scripts/deploy.sh` 进行全量远程部署（至 `mini`）。
   - 校验部署脚本退出状态，确认 `pm2 reload` 成功执行。

2. **主动浏览器验证 (Remote-First Validation)**:
   - **严禁**向用户询问“是否需要验证浏览效果”或在本地 `localhost` 进行测验。
   - 部署完成后，Agent **必须立即、主动**启动 Browser Subagent，访问 `https://opware.kineraw.com` (或受本次修改影响的具体页面)。
   - 若遇需要登录的页面，使用管理员凭证：`admin` / `admin123`。
   - 在线上环境中，根据本次开发的内容主动进行 UI/UX 及关键业务流转的测试。

3. **版本证明与汇报**:
   - 验证页面右上角呈现的绿色软件版本号，确认其与第1步中提交的最新版本号完全一致。
   - 验证结束后，向 Jihua 汇报验证成功的结论，并明确附上**验证通过的版本号 (vX.Y.Z)** 与验证过程发现的核心现象。若验证失败，则直接分析日志并进入下一次迭代，无需结束任务。
