## 引言
本文件面向 Longhorn 前端团队与相关利益方，系统化梳理基于 React 19 + TypeScript + Vite 的现代前端技术栈架构。重点覆盖组件层次结构、状态管理模式（Zustand）、路由配置与国际化支持；阐述前端与后端 API 的交互模式、错误处理机制与缓存策略；总结组件生命周期管理、性能优化技术与开发工具链配置；并给出部署配置、构建优化与生产环境最佳实践，以及跨平台兼容性与移动端适配策略。

**更新** 本次更新重点集成了全新的服务导航架构、通知中心系统、SLA可视化组件和工作台功能，显著提升了服务模块的用户体验和管理效率。客户端版本已从 12.3.107 升级到 12.3.109。

## 项目结构
前端位于 client 目录，采用按功能域分层的组织方式：
- 根入口与构建配置：main.tsx、index.html、vite.config.ts、package.json、tsconfig.json、eslint.config.js
- 应用主体：App.tsx（路由与布局）
- 组件层：components/（业务组件，如 FileBrowser、Login、DailyWord、ServiceRecords、Issues、AppRail、AttachmentZone、TicketCreationModal、TicketAiWizard、ServiceNavigation、WorkspacePage、SlaComponents、NotificationCenter、NotificationBell 等）
- 状态管理：store/（Zustand 状态容器，如 useAuthStore、useDailyWordStore、useTicketStore、useToast、useConfirm、useNotificationStore）
- 国际化：i18n/（翻译字典与语言钩子）
- 工具函数：utils/（路径翻译、日期本地化等）
- 钩子：hooks/（业务逻辑封装，如 useCachedFiles、useCachedTickets、useNavigationState）
- 数据层：data/（词汇数据、配置等）
- 样式层：index.css（统一的 CSS 变量和样式系统）

## 核心组件
- 路由与布局：App.tsx 定义公共路由、受保护路由与主布局 MainLayout/Sidebar/TopBar，实现角色驱动的菜单与权限控制。
- **现代化服务导航**：ServiceNavigation 提供统一的服务模块导航架构，支持折叠分组、活跃状态高亮、徽章计数和本地状态持久化。
- **通知中心系统**：NotificationCenter 提供 macOS 风格的通知面板，支持点击外部关闭、标记已读、批量操作和类型化通知。
- **SLA可视化组件**：SlaComponents 提供优先级标签、SLA状态标签、实时倒计时和状态芯片，支持完整的 SLA 生命周期管理。
- **工作台执行台**：WorkspacePage 提供个人执行台功能，支持工单聚合、筛选排序、SLA预警和交互式工单卡片。
- **服务概览仪表盘**：ServiceOverviewPage 提供管理仪表盘，支持团队负载分析、风险工单监控和 SLA 健康度指标。
- **通知铃铛集成**：NotificationBell 集成到顶部栏，提供服务模块专用的通知入口，支持未读计数轮询和动画效果。
- **现代化布局组件**：AppRail 提供横向应用栏，支持服务模块和文件模块的快速切换，替代传统垂直侧边栏。
- **AI智能工单助手**：TicketAiWizard 提供AI驱动的工单信息提取和表单填充功能，支持从邮件/聊天记录自动解析工单信息。
- **工单创建系统**：TicketCreationModal 提供统一的工单创建界面，支持咨询工单、RMA工单和经销商维修单三种类型。
- **附件上传组件**：AttachmentZone 提供拖拽式文件上传，支持图片、视频、PDF等多种格式，具备预览和删除功能。
- **导航状态管理**：useNavigationState 管理模块切换和路径记忆，支持服务模块和文件模块的智能导航。
- **认证与会话**：useAuthStore 提供用户信息与令牌的持久化存储与登出流程。
- **国际化**：useLanguage 提供语言切换、事件通知与翻译函数；translations.ts 提供多语言字典。
- **通知与确认**：useToast 与 useConfirm 提供轻量级全局提示与确认对话框。
- **文件浏览与缓存**：FileBrowser 使用 useCachedFiles 基于 SWR 实现目录列表缓存、去重与预取。
- **工单缓存架构**：useCachedTickets 基于 SWR 实现工单列表的高效缓存、去重与预取。
- **登录页**：Login 组件负责认证请求与错误展示。
- **每日一词组件**：DailyWordBadge 和 DailyWordModal 提供词汇学习功能，支持多语言、多级别、语音播放等特性。
- **服务记录模块**：ServiceRecordListPage、ServiceRecordCreatePage、ServiceRecordDetailPage 提供客户服务记录管理功能。
- **工单模块**：IssueListPage、IssueCreatePage、IssueDetailPage 提供工单管理系统，支持评论、附件和状态管理。
- **上下文查询**：ContextPanel 提供基于客户或产品序列号的综合查询功能。
- **管理设置**：AdminSettings 提供AI配置和系统管理功能，统一使用 Kine Green 颜色标识。

## 架构总览
前端采用"单页应用 + 现代构建工具"的架构：
- 技术栈：React 19（函数组件 + Hooks）、TypeScript、Vite（开发服务器、代理与构建）
- 状态管理：Zustand（轻量、易用、可序列化），用于认证、通知、确认对话框、每日一词状态、工单草稿状态、通知状态管理
- 数据获取：SWR（缓存、去重、刷新、预取），结合 axios 进行 API 请求
- 国际化：自研 useLanguage + translations 字典，支持 zh/en/de/ja
- 路由：react-router-dom 7（嵌套路由、Outlet、受保护路由）
- 样式与主题：CSS 变量与 Tailwind 相关类名（通过样式变量实现暗色主题），统一的颜色标准化系统

**更新** 新增服务导航架构、通知中心系统、SLA可视化组件和工作台功能，形成完整的服务模块生态系统。客户端版本已升级至 12.3.109。

### 服务导航组件（ServiceNavigation）
**新增** 统一的服务模块导航架构，支持折叠分组、活跃状态高亮、徽章计数

- **设计理念**：基于 macOS26 风格的玻璃拟态设计，提供清晰的模块分组和视觉层次
- **核心功能**：
  - 折叠分组：支持工作台、档案和基础信息、技术中心、管理等分组
  - 活跃状态：根据当前路径自动高亮对应导航项
  - 徽章计数：支持未读数量显示，使用红色圆形徽章
  - 本地状态：使用 localStorage 持久化展开状态
  - 特殊处理：wiki 和 bokeh 路径的特殊匹配逻辑
- **交互设计**：悬停效果、激活状态、平滑过渡动画，提供丰富的视觉反馈
- **图标系统**：使用 lucide-react 图标库，统一的图标风格和尺寸

### SLA可视化组件（SlaComponents）
**新增** 完整的 SLA 生命周期管理组件系统

## 性能考量
- 缓存与去重：SWR 的 dedupingInterval 与 keepPreviousData 提升导航流畅度；ETag 使轮询成本较低
- 预取策略：prefetchDirectories 与组件内预取减少二次打开延迟
- 状态管理：Zustand 无样板代码，避免不必要的重渲染；Toast/Confirm 采用最小必要状态
- **每日一词优化**：词汇批处理加载（100个词汇），本地存储配置，避免重复网络请求；新增缓存重置功能提升用户体验
