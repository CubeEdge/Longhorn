## 项目概述

Longhorn 是一个企业级服务工作流管理系统，专注于增强服务工单处理流程。该项目实现了完整的 P2 架构升级，采用统一工单模型设计，支持三种工单类型：咨询工单（Inquiry）、RMA 工单（RMA）和经销商维修工单（DealerRepair）。

系统的核心特性包括：
- **统一工单架构**：单表多态设计，支持三种工单类型的统一管理
- **智能工作流**：基于部门职责的自动化分发机制
- **可视化仪表盘**：为管理层提供实时的工单状态监控
- **三视图工作台**：个人任务、提及任务和部门队列的统一界面
- **自动分发规则**：基于部门和节点的智能工单分配
- **增强的列排序系统**：支持按优先级、ID、标题、状态、SLA 期限排序
- **灵活的列宽调整**：用户可自定义列宽布局
- **优化的视觉系统**：改进的优先级徽章和星标系统
- **专业文档工作流**：完整的PI和维修报告管理流程
- **商务审核机制**：集成保修计算和费用确认流程
- **费用结算系统**：支持保内外维修的费用结算和流转控制
- **三层次产品架构**：支持产品目录、商品规格和设备台账的完整管理体系
- **增强的保修计算**：实现五级水位线计算和损坏拦截机制
- **改进的交接流程**：强化文档发布前置检查和结案确认机制
- **运营维修报告编辑器**：支持OP部门的维修记录编辑和管理
- **统一SN状态驱动工作流系统**：基于序列号状态的智能工单流转
- **增强活动修正系统**：支持对已提交活动的精确字段更正和审计追踪
- **UI/UX深度重构**：侧滑窗标准化、表单优化和时间轴数据密度优化
- **数据库迁移增强**：产品架构升级、活动编辑列和报告者快照等
- **工作流推进增强**：新增更正模式检测逻辑，防止对已完成阶段的工单进行工作流推进
- **结案结算功能增强**：FinalSettlementModal 支持保内外维修处理和费用结算
- **关键节点更正权限控制**：确保只有授权人员可修改关键节点
- **PDF导出设置增强**：PIEditor和RepairReportEditor支持格式、方向、多语言、页眉页脚自定义
- **财务计算和成本跟踪增强**：实时费用汇总和准确的成本计算

#### SN状态检测机制

系统支持多种序列号状态的自动检测：

- **激活状态** (`activated`)：设备首次激活时自动创建RMA工单
- **维修中状态** (`in_repair`)：设备进入维修流程时创建维修工单
- **返厂状态** (`return_to_factory`)：设备需要返厂维修时创建返厂工单

2. **维修中状态**：自动分配给OP运营部门的维修技师
3. **返厂状态**：自动分配给物流部门的处理人员
4. **报废状态**：自动分配给报废处理部门

#### 状态流转优化

系统根据SN状态自动优化工单状态流转：

- **RMA流程**：从 `op_receiving` → `op_diagnosing` → `ms_review` → `op_repairing` → `ms_closing` → `op_shipping` → `resolved`
- **维修流程**：从 `op_receiving` → `op_diagnosing` → `op_repairing` → `op_qa` → `ms_closing` → `op_shipping` → `resolved`
- **返厂流程**：从 `op_shipping` → `shipped` → `resolved`

## 性能考虑

### 数据库优化策略

1. **索引优化**：为常用查询字段建立索引，包括 `ticket_type`、`current_node`、`assigned_to`、`status` 等
2. **查询缓存**：对频繁访问的统计数据使用内存缓存
3. **分页处理**：默认每页200条记录，避免大数据量查询
4. **连接池管理**：使用 better-sqlite3 的连接池机制
5. **文档序列号缓存**：PI和维修报告序列号独立缓存，避免重复查询
6. **产品架构索引**：为产品模型、商品规格和产品实例表添加适当索引
7. **保修计算缓存**：保修计算结果在内存中缓存，避免重复计算
8. **自动保存优化**：运营维修报告编辑器使用5秒防抖，减少API调用频率
9. **缩略图缓存**：附件缩略图在磁盘和内存中双重缓存
10. **活动修正审计**：更正历史记录使用增量更新，避免全量重写
11. **SLA计算优化**：SLA状态计算使用缓存机制，避免重复计算
12. **权限管理优化**：文件权限使用哈希索引，提升查询性能
13. **序列号状态优化**：SN状态检测使用索引优化，提升响应速度
14. **工作流推进优化**：更正模式检测使用前端状态缓存，减少API调用
15. **结案结算优化**：结算数据使用本地缓存，避免重复计算
16. **关键节点权限优化**：权限检查使用本地缓存，提升响应速度
17. **PDF导出优化**：客户端渲染，避免服务端压力
18. **财务计算优化**：实时计算使用防抖处理，减少计算开销
19. **自动保存优化**：防抖机制减少数据库写入频率
20. **版本控制优化**：文档版本使用增量更新，避免全量复制

### 前端性能优化

1. **虚拟滚动**：对长列表使用虚拟滚动技术
2. **懒加载**：组件按需加载，减少初始包大小
3. **状态持久化**：使用 localStorage 缓存用户偏好设置
4. **防抖处理**：搜索和筛选操作使用防抖机制
5. **列宽缓存**：使用 localStorage 缓存列宽设置
