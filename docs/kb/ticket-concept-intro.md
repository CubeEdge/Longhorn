# 工单概念入门

在 Longhorn 系统中，工单并不是杂乱堆放的。通过 **"Workspace(工作空间)"** 侧边栏，我们将工单按**责任归属**进行了精细化分类。

## 1. 我的任务 (My Tasks)

这是您的第一个核心视图，相当于您的 **"个人待办列表"**。

*   **包含内容**：所有 **Assignee (负责人) 为您本人** 的工单。
*   **用途**：
    *   专注于正在处理中的客户咨询或维修任务。
    *   绝大部分流转动作（传球）都在此处完成。
*   **球的位置**：球就在您的工位上，所有人都能看到您是当前的操作人。

---

## 2. 团队协作 (Team Hub)

这是部门内部的 **"公共任务池 (Public Pool)"**。

*   **包含内容**：所有 **Assignee (负责人) 为 NULL** 的工单。这些工单通常处于 `Submitted` (已提交) 或 `op_diagnosing` (待诊断) 等公共节点上，正等待有人去处理。
*   **用途**：
    *   **抢单/分配**：您可以在此查看所有待处理的工单，点击 **[ 认领 / Claim ]** 将其移至您的 "My Tasks"。
    *   **监控工作负载**：Lead 可以通过这个池子判断部门是否有积压。
*   **球的位置**：球在部门门口的篮子里，由大家共管。

---

## 3. 部门工单 (Dept Queue)

这是部门的 **"全家桶 (Full View)"**。

*   **包含内容**：所属部门的所有工单，无论是否已分配、谁是负责人。
*   **用途**：
    *   协助其他成员处理工单。
    *   查找部门内过往的记录。
    *   Lead 用来宏观把控部门所有进行中的任务。
*   **球的位置**：全局概览，球在谁手里一目了然。

---

## 4. 回收站 (Recycle Bin / Trash)

这是工单的 **"缓冲区"**，目前仅 **Admin** 或 **MS Lead** 有权查看。

*   **进入方式**：在 `Team Hub` 视图右上角点击 `...` (更多) 按钮。
*   **包含内容**：处于 `Archived` 状态的逻辑删除工单。
*   **用途**：
    *   防止误删。
    *   管理角色可以点击 **[ 恢复 / Restore ]** 将工单拉回正常状态，或执行物理删除。
*   **球的位置**：在废弃篮中，不可被普通成员搜索到。

---

## 💡 总结

| 视图 | 核心价值 | 适用场景 |
| :--- | :--- | :--- |
| **我的任务** | **专注** | 快速处理自己手头的活 |
| **团队协作** | **共进** | 认领新任务，支援堆积环节 |
| **部门工单** | **透明** | 查漏补缺，宏观统计，跨员协助 |
| **回收站** | **避险** | 数据资产的最后防线 |
