# 工单权限与操作：谁能编辑/删除？

为了平衡**处理效率**与**风险控制**，系统采用了**阶梯式风险控制模型**。本篇文档详细说明各角色在不同阶段对工单的操作权限。

## 1. 核心权限原则：级联放权与锁定

*   **活跃期放权**：在工单处理过程中，赋予相关人足够的编辑权。防止流程卡死。
*   **结算期锁定**：工单进入审核、终结状态后，对普通角色置灰，仅允许特权角色修正。
*   **归属于审计**：非特权用户的操作必须与其身份挂钩（是负责人或创建者），且核心字段修改必须强制填写原因。

---

## 2. 角色权限详情

### A. 管理角色 (MS Lead / Admin / Exec)

**特权定义**：拥有"超级编辑"与"物理删除"权。

*   **可见性**：始终能看到【编辑】按钮、三点菜单中的【废弃/删除】按钮。
*   **入口**：Team Hub 右侧设有【...】（更多）按钮，进入**回收站**查看已删除工单。
*   **恢复权**：只有该角色能从回收站恢复误删工单。

### B. 普通处理人 / 其他部门人员 (OP Lead, MS Member 等)

**权限限制**：基于"归属关系"的有限操作。

*   **编辑权条件**（需同时满足）：
    1. 工单处于**活跃阶段**（非 Resolved/Closed 等）。
    2. **身份匹配**：必须是该工单的 **Assignee (负责人)**、**Reporter (创建者)** 或 **Participant (参与者)**。
*   **删除权条件**（需同时满足）：
    1. 工单处于**极早期**（Draft 或 Submitted）。
    2. **身份匹配**：必须是该工单的 **Reporter (创建者)**。

---

## 3. UI 入口守卫逻辑

系统通过"可见性守卫"确保用户只能看到能操作的功能，不存在"点了报错"的挫败感：

1.  **编辑按钮**：若不满足上述条件，详情页右上角将完全隐藏【编辑】。对于 Admin 进行 View As 预览时，系统会自动应用被模拟用户的权限。
2.  **删除菜单**：只有具备删除权限的人，详情页右上角才会出现【...】三点菜单。
3.  **回收站**：只有 MS Lead 或 Admin 在 Team Hub 视图下才能看到三点菜单入口进入回收站。

---

## 4. 强制审计 (Audit Whitelist)

修改以下字段时，系统会要求填写"修改原因"：

*   **设备/归属**：序列号、产品型号、关联账号/联系人/经销商。
*   **核心内容**：问题简述、报修详述、维修内容。
*   **经济/时效**：保修判定、金额、优先级、SLA截止时间。
