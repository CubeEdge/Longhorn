## 0. 三层工单模型概述

本系统采用三层工单模型：

| 工单类型 | ID格式 | 示例 | 用途 |
|---------|--------|------|------|
| **咨询工单** | KYYMM-XXXX | K2602-0001 | 客户咨询、问题排查、远程协助等 |
| **RMA返厂单** | RMA-{C}-YYMM-XXXX | RMA-D-2602-0001 | 设备寄回Kinefinity总部维修，每台设备一个RMA号 |
| **经销商维修单** | SVC-D-YYMM-XXXX | SVC-D-2602-0001 | 经销商本地维修，用于配件消耗和库存管理 |

**渠道代码 {C}**：
- D = Dealer (经销商)
- C = Customer (直客)
- I = Internal (内部)

**系统开放阶段**：
| 阶段 | 开放对象 | 咨询工单 | RMA返厂单 | 经销商维修单 |
|-----|---------|---------|----------|-------------|
| **1.0** | 市场部 | 市场部创建 | 市场部代创建 | 市场部代创建 |
| **2.0** | 经销商 | 经销商可创建 | 经销商提交需审批 | 经销商自行管理 |
| **3.0** | 终端客户 | 客户可提交 | - | - |

## 1. 用户角色定义

### 1.1 当前系统用户范围

| 角色 | 代表人物/示例 | 使用端 | 主要职责 |
|-----|-------------|-------|---------|

| **一级经销商** | ProAV(英国), Gafpa(荷兰), 1SV(美国), EU Office(荷兰) | Web | 代客户反馈、申请RMA、本地维修、配件库存管理 |
| **二级经销商** | DP Gadget(东南亚) | Web | 代客户反馈、申请RMA、本地维修(需临时发配件) |
| **三级经销商** | RMK(澳洲) | Web | 代客户反馈、申请RMA、协调寄回总部 |

> 注：经销商暂时仅支持Web端，未来版本考虑iOS端

### 1.2 经销商分级说明

| 分级 | 特点 | 配件库存 | 维修能力 | 结算方式 |
|-----|------|---------|---------|---------|
| **一级** | 区域分销商，销售能力强 | 有，定期补货 | 强 | 定期结算 |
| **二级** | 有维修能力，销售能力一般 | 无，需临时发货 | 有 | 单次开PI |
| **三级** | 无维修能力，销售能力一般 | 无 | 无 | 单次开PI |

**一级经销商** (ProAV, Gafpa, 1SV, EU Office):
- 可处理大部分不涉及刷程序、清灰的维修
- 复杂问题需寄回总部

**二级经销商** (DP Gadget):
- 有维修能力但无备件，需Kine临时发配件
- 一般问题可本地维修，复杂问题需寄回总部

**三级经销商** (RMK):
- 无维修能力、无备件
- 所有问题均需寄回总部维修

### 1.3 暂不开放的用户

## 2. 市场部使用场景

> **核心职责**：市场部是系统的核心使用者，承担客户问题接收、工单协调、经销商对接、生产部协作等职能。

**咨询工单生命周期**:
```
创建 → 处理中 → 待客户反馈 ─┬─ 客户确认 → 已解决
                          ├─ 7天无回复 → 自动关闭
                          ├─ 客户回复继续 → 回到处理中
                          └─ 需要维修 → 升级为RMA/SVC

自动关闭规则:
• 第5天: 发送提醒"您的服务请求即将关闭"
• 第7天: 自动关闭

重新打开规则:
• 30天内同问题 → 重新打开原工单
• 超过30天或新问题 → 创建新工单
```

**角色**: 市场部 / 经销商 / 管理层
**触发**: 在与 Bokeh 对话过程中，AI 识别出技术支持意图并检索到高相关性的历史工单。

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 用户提问: "Edge 8K 录制 4K 50fps 死机了，有类似案例吗？"   │
└─────────────────────────────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. Bokeh 自动执行背景任务:                                    │
│    - 提取关键词: "Edge 8K", "4K 50fps", "死机"              │
│    - 检索 FTS5 索引库，发现 Top-3 相关结案工单                │
│    - 校验用户权限: 经销商 A 仅能看到 A 处理过的历史工单       │
└─────────────────────────────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. Bokeh 生成回复 (带链接):                                   │
│    "对不起，我找到了以下类似案例：                          │
│     - [K2602-0001] 🎫咨询: MAVO Edge 8K 拍摄死机问题         │
│     - [SVC-D-2601-0015] 🛠️经销商: 4K 模式死机排查            │
│     建议：升级固件至 8023，并检查机身散热。"                  │
└─────────────────────────────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. 用户交互 (详情查看):                                       │
│    - 用户点击 [K2602-0001] 链接                               │
│    - 系统弹出工单详情弹窗 (Popup)，显示该工单的最终解决方案     │

└─────────────────────────────────────────────────────────────┘
```

**关键体验**:
- **无缝衔接**: AI 不再只是“聊天”，而是直接桥接了系统内的原始数据。
- **权限安全**: 强制性的 Dealer ID 过滤确保敏感维修信息不会外泄给非关联方。
- **玻璃拟态 UI**: 详情弹窗保持 Bokeh 的轻盈感，不破坏当前的对话上下文。
