# 产品管理命名重构方案

## 问题背景

当前系统中存在命名混淆问题：
- "产品管理" 实际管理的是 **Installed Base（设备台账）**，即每一台出厂的具体设备（带唯一SN）
- "产品型号" 管理的是 **Product Catalog（产品目录）**，即产品型号定义

两者界面相似但用途完全不同，导致用户困惑。

## 重构方案

### 1. 重命名明确区分

| 当前名称 | 建议名称 | 说明 |
|---------|---------|------|
| 产品管理 | **设备台账** / Installed Base | 管理具体出厂设备，含SN、客户、保修信息 |
| 产品型号 | **产品目录** / 型号管理 | 管理产品型号定义、规格参数 |

### 2. 导航结构调整

```
服务管理
├── 设备台账（原产品管理）- 管理出厂设备SN
├── 产品目录（原产品型号）- 管理型号定义
└── 配件管理
```

### 3. 界面差异化

**设备台账**：
- 突出字段：SN、客户信息、保修状态、维修历史
- 搜索维度：SN、客户名称、型号

**产品目录**：
- 突出字段：规格参数、图片、文档
- 搜索维度：型号名称、产品系列

### 4. 双向关联入口

- 设备台账页面：添加 [管理型号定义] 快捷入口
- 产品目录页面：添加 [查看此型号设备] 统计链接

## 实施优先级

1. **高优先级**：重命名菜单和页面标题
2. **中优先级**：调整导航结构
3. **低优先级**：界面字段差异化展示
